# Halal Damu Bot - Техникалық Архитектурасы

## 1. Технологиялық стек (Tech Stack)
- **Frontend (Клиенттік бөлік):** React 18, TypeScript, Tailwind CSS, Vite.
  - *Пайдаланушы интерфейсі:* `shadcn/ui` компоненттері.
  - *Маршрутизатор:* `react-router-dom`.
- **Backend (Серверлік бөлік):** Node.js, Express.js.
  - *Логика:* API эндпоинттері (`/api/admin/...`), Telegram (бұдан былайғы байланыстар).
- **Деректер қоры (Database):** Google Cloud Firestore (NoSQL).
- **Хостинг / Инфрақұрылым:** Google Cloud Run (Docker контейнері арқылы).

## 2. Қосымшаның құрылымы (App Structure)
- `src/App.tsx`: Негізгі клиенттік интерфейс (Dashboard), "Terminal" және "CRM" бөлімдері.
- `src/pages/AdminCMS.tsx`: Деректерді басқару панелі (Мекемелер мен қоспаларды көру, өңдеу, өшіру, пейджинация).
- `server.ts`: Express серверінің негізгі файлы. Маршруттар, API және бот инициализациясы осында анықталған.
- `src/server/src_server_db.ts`: Firestore деректер қорына қосылу және деректер кэшін басқару конфигурациясы.
- `src/server/src_server_search.ts`: Кешенді іздеу логикасы (Е-кодтарға және мекемелерге Python нұсқасындағыдай толық 1:1 алгоритммен іздеу: сөздік транслитерациялау нұсқалары, тоқтату сөздерін алып тастау, fuzzball partial_ratio мен ratio-ның нақты коэффициенттері және геолокация бойынша сәйкестендіру).
- `src/server/src_server_bot.ts`: Телеграм боттың негізгі бақылау файлы.
- `src/server/` Телеграм боттың және скрипттердің жалпақ жинақталған архитектурасы (Flat Single-Directory Architecture, Cloud Run Functions үшін папкасыз тікелей орналастырылатын пішім):
  - `src_server_bot_types.ts`: Сессия мен бот контекстінің интерфейстері.
  - `src_server_bot_middlewares.ts`: Форум/Супертоп тақырыптарын қолдау, сессия қауіпсіздігі және қолданушы белсенділігін тіркеу утилиттері.
  - `src_server_bot_helpers.ts`: Пайдаланушыға жауап беру безендірулері, Карта батырмалары және беттеу (pagination) визуализаторлары.
  - `src_server_bot_handlers_text.ts`: Мәтінді талдау, AI бағыттауыш және дерекқор іздеу өңдеушісі.
  - `src_server_bot_handlers_location.ts`: 10 км радиустағы халал кафелерді табу утилиті.
  - `src_server_bot_handlers_photo.ts`: Суретті AI арқылы талдау.
  - `src_server_bot_handlers_callbacks.ts`: Орындалатын кері байланыс және батырмаларды басқару.
  - `src_server_bot_streamUtils.ts`: Телеграмдағы прогрессивті стриминг утилиті.
  - `src_server_bot_topicRenamer.ts`: Gemini арқылы тақырып атауларын ақылды басқару.
  - `src_server_bot_intentClassifier.ts`: Пайдаланушы хабарламасын сөйлесу (chat) немесе іздеу (search) деп жіктейтін AI модулі.
  - `src_server_bot_aiChat.ts`: AI-мен еркін сөйлесу және табылмаған жағдайдағы сыпайы жауаптар генераторы.
  - `src_server_scripts_sync_companies.ts`: Сыртқы API арқылы мекемелерді синхрондау скрипті.
  - `src_server_scripts_migrate_ingredients.ts`: Тағамдық қоспаларды миграциялау скрипті.
  - `src_server_utils.ts`: Жалпы көмекші утилиттер форматы.
  - `src_server_quotes.ts`, `src_server_quotesKazakh.ts`, `src_server_quotesQuran.ts`: Деректер дизайны үшін дәйексөздер қорлары.
  - `src_server_agenticImageSearch.ts`: Суретті талдап, дерекқордан іздейтін модуль.
- `src/server/src_server_migrate_phase2.ts`: Деректерді жаппай жаңартуға/миграция жасауға арналған скрипт.

## 3. Деректер қорының құрылымы (Database Schema)

### Жинақ: `search_ingredients` (Тағамдық қоспалар / Е-кодтар)
- **Идентификатор (ID):** Тағамдық қоспаның Е-коды немесе кілттік атауы (мысалы, `e102`, `e120` т.б.).
- **Өрістер:**
  - `id` (String): Жүйелік идентификаторы (мысалы, "E102").
  - `name_kz`, `name_ru` (String): Қоспаның қазақша және орысша атаулары.
  - `type` (String): Қоспаның функционалды түрі (бояғыш, эмульгатор, консервант т.б.).
  - `source_type` (String): Түпкі шығу тегі (мысалы, "synthetic", "animal", "plant", "microbiology" т.б.).
  - `status` (Object): Халал мәртебесі:
    - `id` (String - "halal", "haram", "mushbooh").
    - `title` (String - қазақша мәртебе атауы).
    - `reason` (String - халал немесе харам болу себебінің анықтамасы).
    - `level` (Number - қауіп деңгейі).
  - `is_active` (Boolean): Қоспаның іздеу нәтижесінде белсенді болу баптауы.

### Жинақ: `search_companies` (Сертификатталған мекемелер мен өндірушілер)
- **Идентификатор (ID):** Автоматты түрде құрылатын Firestore құжат идентификаторы.
- **Өрістер:**
  - `title`, `legal_name`, `brand` (String): Компанияның ресми атауы, заңды тұлға аты және бренді.
  - `category` (String): Негізгі қызмет категориясы (мысалы, "Мейрамхана", "Өндіруші", "Қоғамдық тамақтану" т.б.).
  - `status` (Object): Халал сертификатының ағымдағы статусы (мысалы ID: "active", title: "Сертификатталған").
  - `certificate` (Object):
    - `status` (String - "active" немесе "expired").
    - `issued_at`, `expires_at` (String - ISO 8601 уақыт форматы).
    - `number` (String - сертификат реттік нөмірі).
  - `address`, `map_link`, `city` (String): Мекенжайы, Карта сілтемесі және орналасқан қаласы.
  - `description_clean`, `short_description` (String): Кәсіпорын сипаттамасы мен қысқаша мәліметі.
  - `is_active`, `is_delivery_only`, `has_prayer_room`, `has_halal_alcohol` (Boolean): Белсенділік белгілері мен қосымша жағдайлар (намазхана бар ма, алкогольсіз бе т.б.).
  - `title_aliases`, `tags` (Array): Іздеу транслитерациясы және сәйкестендіруге арналған синонимдік тіркестер/тегтер жиыны.
  - `email`, `website`, `logo_image` (String): Байланыс деректері және логотип сілтемесі.

### Жинақ: `chat_history` (Пайдаланушылар чат тарихы)
- **Идентификатор (ID):** Firestore-да құжат атауы ретінде пайдаланушының Telegram ID-і қолданылады.
- **Өрістер:**
  - `userId` (Number): Пайдаланушының Telegram ID-і.
  - `history` (Array): Сақталған әңгімелер фрагменттері. Әрбір фрагмент келесі форматта болады:
    - `role` (String): "user" немесе "model".
    - `parts` (Array of objects): Текстік мәліметі бар жиынтық (`[{ text: string }]`).
  - `updatedAt` (Timestamp): Соңғы рет жаңартылған уақыт белгісі.

## 4. Деректер алмасу логикасы (Data Flow)
1. **Frontend -> Backend:**
   CMS-те деректер (мысалы, "Сақтау" батырмасы) өзгертілген кезде `PUT /api/admin/search_companies/:id` немесе `PUT /api/admin/search_ingredients/:id` бағыты бойынша JSON payload жіберіледі.
2. **Backend -> Firestore:**
   Express сервері деректерді қабылдап, валидация жасайды (мысалы, `tags` немесе `aliases` жиым екенін тексереді) және `docRef.update()` арқылы базаға жазады.
3. **Backend -> Frontend:**
   Сервер `200 OK` мен жаңартылған нысанды қайтарады. Клиент `AdminCMS` компонентінде күйді (state) жаңартып, өзгерісті экранда тікелей көрсетеді.
4. **Мекемелерді сыртқы API-ден синхрондау (Companies Sync Engine):**
   - Күнделікті сағат 03:00-де (Азия/Алматы уақытымен) Express серверінде автоматты түрде `node-cron` арқылы синхрондау скрипті іске қосылады.
   - Скрипт `https://halaldamu.kz/wp-json/halal-bot/v1/companies?lang=kz` мекенжайындағы қазақ тіліндегі ресми API-дан жаңа деректерді алады.
   - Дерекқордағы ескі мекемелер Firestore Batch Delete арқылы толық тазартылып, жаңа қазақтілді мекемелер жиынтығы Firestore Batch Write (әр партияда 300 нысаннан топтастырылып) арқылы жазылады.
   - Жазылып біткен соң, Express серверіндегі іздеу жады (In-Memory Cache) автоматты түрде жаңартылады, бұл пайдаланушыларға іздеу нәтижелерін лезде және тек қазақ тіліндегі нұсқада алуға кепілдік береді.

## 5. Суретті тану архитектурасы және AI Клиент (Image Recognition & Fast OCR/Brand Extraction Architecture)

Жүйе сурет арқылы өнімді, брендті немесе тағамдық құрамды талдап, дерекқордан лезде тауып беру үшін **gemini-flash-lite-latest моделінің тездетілген бір қадамдық JSON (Single-turn JSON output) режімін** және орталықтандырылған AI Клиентті қолданады:
- **Басымдылық ретіндегі Vertex AI (Vertex AI Priority Flow):** Егер жүйеде GCP Service Account конфигурациясы (`GOOGLE_CREDENTIALS_JSON`) бар болса, бот барлық сұраныстарда (мәтіндік генерация, стриминг және эмбеддинг векторларында) Google Cloud Vertex AI-ды тікелей негізгі басымдылықпен (Primary Client) қолданады. AI Studio (API Key) тек қорғаныстық қосалқы (fallback Engine) ретінде артқы фонда қалады. Бұл квота таусылу немесе timeout қателіктерін толық жояды.
- **Орталықтандырылған Сурет Талдау (Consolidated Image Search):** Сурет талдау жүйесі (`agenticImageSearch.ts`) өзінің жеке GoogleGenAI данасын құрмайды, ол да осы басты `aiClient` қорғаныс торынан шақырылып, Vertex AI басымдылықтарына бірден бағынады.
- **Сенімді және Жетілдірілген Әрекет Көрсету (Robust Native Chat Action with sendChatAction):** Пайдаланушыларға жауапты дайындау барысында фантомдық қателіктер мен 90,000ms тайм-ауттарды болдырмау үшін Telegram-ның стандартты `sendChatAction("typing")` утилиті қолданылады. Бұл ресми Telegram Bot API серверлерінде қателіктерсіз әрі өте жылдам жұмыс істеуге кепілдік береді. Бизнес-қосылымдар мен топтық чаттарда кез келген байланыс мәселесін жойып, боттың жауап жылдамдығын максималды арттырады.
- **Тақырыптарды ақылды өзгерту (Daraq-Style Smart Renamer):** Топтар мен тақырыптарда тақырып атауын автоматты түрде өзгерту логикасы **Daraq** архитектурасына сәйкес жетілдірілді. Егер пайдаланушымен жәй сөйлесу (chitchat/conversational) орын алса, топ шуламауы және артық өзгерістер болмауы үшін тақырып атауы өзгеріссіз қалады. Ал егер нақты іздеу (knowledge/product search), маңызды сұраныс, суретті AI арқылы талдау немесе геолокациялық (орын) сұраныс жіберілсе, жүйе оны Gemini LLM көмегімен қысқа әрі нұсқа (эмодзи қоса отырып, мысалы сурет үшін сәйкес анықтама мен орын үшін "📍 Айналадағы жақын орындар/шешімдер") ретінде автоматты түрде өзгертіп отырады.
- **Автоматты HTML тегтерін жабу тетігі (Auto HTML Tag Closer):** Стриминг кезінде Telegram-ның "mismatched entities" (ашық қалған тегтер) қателіктерін болдырмау үшін әр хабарлама жіберілу алдынде барлық ашық қалған HTML тегтері (`<b>`, `<i>`, `<code>`) автоматты түрде өзінің жабылу таңбаларымен жабылады. Бұл стримингтің үзіліссіз, біртіндеп, әдемі шығуын қамтамасыз етеді.
- **Маркдауннан тазарту (Robust MD-to-HTML Sanitation):** Жауап барысында пайда болатын `**`, `##`, `__` секілді маркдаун белгілері мен артық жұлдызшалар автоматты түрде тазартылып, таза HTML форматында беріледі.

## 6. Ерекше мүмкіндіктер (Key Features)

- **Икемді маршрутизатор (Intelligent Router & Classifier):** Бұл маршрутизатор **Daraq** архитектурасындағы репо логикасымен одан әрі жетілдірілді. Код ішіндегі барлық қолдан тізілген кілт сөздер сүзгілері толығымен жойылып, адамдардан келген кез келген хабарлама КҮМӘНСІЗ бірінші болып Gemini-дің интеллектуалды жіктеуіш моделіне бағытталады. Модель хабарламаны оқып БІРДЕН жауап ретінде әрекетті (`chat` немесе `search`) және сәйкес шешімді JSON форматында қайтарады. Егер модель сұранысты жай сөйлесу (`chat`) деп таныса, ол екінші рет LLM шақыруын күтпей, **бірден дайын жауап мәтінін қоса дайындап шығарады**. Ал егер нақты іздеу сұранысы қойылса (`search`), модель сұраныстан барлық артық сөздерді алып тастап, негізгі бренд немесе өнім атауын пайдаланушы жазған түпнұсқа түрінде тазартып шығарады, содан соң жүйе оны дерекқордан тікелей әрі қауіпсіз іздейді. Бұл өзара сөйлесулер мен іздеулерде 100% дәлдікті қамтамасыз етеді, базадан кездейсоқ қате сәйкестіктердің алдын алады.
- **Еркін сөйлесу режімі (Chat Mode with Memory):** Firestore дерекқорында сақталған пайдаланушы чат тарихын (`chat_history`) ескере отырып, Gemini 3.5 өте мәдениетті әрі сыпайы (Сіз деп) халал тұрғысынан ақыл-кеңес береді. Сымбат атты ерекше пайдаланушы үшін арнайы махаббат пен еркелетуге негізделген Easter-Egg режімі құрылған.
- **Кері байланыс жүйесі (Feedback System):** Бұрын қолданылған "👍 Пайдалы" және "👎 Қате" кері байланыс батырмалары чат интерфейсін таза әрі жинақы сақтау үшін пайдаланушылардың сұрауы бойынша толықтай жойылды.
- **Дерекқор іздеу сәйкестігі мен батырмалар:**
  - Бірнешеу табылса, оларды интерактивті пейджинация тізімі ретінде таңдауға ұсынады.
  - Е-кодтар табылса, пайдаланушы бір батырмамен әр қоспаның анықтамалығын дәл аша алады.
- **Орналасқан жер батырмасының тұрақтылығы (Button Persistence):** Пайдаланушы әр түрлі сұрақтар қойғанда немесе AI-мен сөйлескенде де, жеке чатта `📍 Айналадағы халал мекемелер` батырмасы ешқашан жоғалып кетпейді, ол барлық жауап хабарламаларында клавиатура ретінде сақталып отырады.
- **Пейджинация (Pagination):** Браузер немесе чат жүктелмеуі үшін тізімдерді бөліп көрсету.
- **Терминал (Terminal):** Жүйенің ішкі логикасы мен логтарын көрсететін бақылау интерфейсі.
- **Икемді форма:** Форманың элементтері тік бағытта (vertical-stack) бір бағанға жиналған, әртүрлі құрылғыларда интуитивно жұмыс істейді.
- **Топтардағы тақырыптарды (Forum Topics) қолдау:** Тікелей супертоптар ішіндегі белгілі бір тақырыптарда (Forum Threads) бот келген хабарламаларға дәл сол тақырыптың (`message_thread_id`) ішінде жауап қайтарады. Бұл Telegraf-тың барлық хабарлама жіберу әдістерін және `ctx.telegram.callApi` прототипін қауіпсіз тұзаққа салу (hook/intercept) арқылы жүзеге асырылған.
  - *Орналасқан жер батырмасы мен үйлесімділік:* Жеке чаттардағы орналасқан жерді бөлісу клавиатурасын сақтау және Telegram Desktop секілді сыртқы клиенттердің текстік ерекшеліктерімен үйлесімділік үшін `📍 Айналадағы халал мекемелер` текстік командасын тікелей өңдеп гео-интерактивті бағыттау мүмкіндігі қорғалған. Пайдаланушыларға ыңғайлы болу үшін негізгі клавиатура қос қатарды (Double-row structure) иеленді: 1-ші қатарда ұялы телефондар үшін GPS-ті бірден жіберетін `📍 Менің орнымды жіберу` (location button request), ал 2-ші қатарда Desktop/нұсқаулыққа арналған `📍 Айналадағы халал мекемелер` мәтіндік батырмасы тұрақты орналастырылды.
  - *Топтық чаттар мен арналардағы автоматты сүзу (Group Mention Filter):* Топтарда боттың басқа пайдаланушылардың өзара әңгімесіне кедергі жасамауы үшін `groupMentionFilterMiddleware` енгізілді. Бот топтық чаттарда тек оны `@username` ретінде тікелей атап өткенде немесе боттың өз хабарламасына Reply (жауап) арқылы жазғанда ғана сұранысқа құлақ асады, әйтпесе хабарламаны елемейді.
  - *Іздеу мәтінін тазарту (Mention Cleaning):* Пайдаланушы топтық чатта бот атауын көрсеткенде (мысалы, `@HalalDamu_bot Шұжық`), сұранысты өңдеу алдында `@username` тіркесі мәтіннен автоматты түрде алынып тасталады. Бұл іздеу жүйесі мен AI өңдегішінің таза іздеу кілттерімен жұмыс істеуіне кепілдік береді.
  - *Топтарда автоматты тақырып құруды өшіру:* Топтық чаттарды ретсіз тақырыптармен толтырмау және топ ішіндегі ыңғайлылық үшін жаңа сұрақтарға автоматты түрде жеке Forum Topic (тақырып) құру логикасы өшірілді. Бот кез келген топтық чатта немесе Forum Topic ішінде өз орнында ғана жауап жазады.

## 7. Боттың іске қосылу режимі (Bot Launching Mode)

Қазіргі нұсқада Telegram бот тек **Long Polling (ұзын сұрау)** режимінде іске қосылады. Бұл кез келген ортада (соның ішінде AI Studio-ның жабық локалды ортасы мен Cloud Run өндірістік контейнерінде) сервердің сыртқы URL мекенжайын тіркеуді қажет етпестен, Telegram серверлерінен хабарламалар мен жаңартуларды автоматты түрде тікелей тартып алуды қамтамасыз етеді және желілік бағыттау мәселелерін немесе Webhook арнасын баптау қажеттілігін толығымен сейілтеді.

- **Ерекшеліктері:**
  - **Масштабталуы:** Тестілеу орталары мен локалды тұғырларда ешқандай конфигурациясыз бірден іске қосылады.
  - **Кезек тазарту (Pending updates cleanup):** Бот іске қосылғанда `dropPendingUpdates: true` параметрі қолданылады. Бұл бот өшіп тұрған немесе жаңартылып жатқан аралықта жиналған ескі хабарламалар кезегін тазартады, осылайша бот қайта қосылғанда ескі хабарламаларды қайталап өңдемейді және жүйелік жүктемені азайтады.
  - **Ақауларды автоматты түзету:** Токенде қайталанған идентификаторлар (ID:ID:SECRET) кездескен жағдайда бот оны ішкі алгоритммен түзетіп барып іске қосылады.
